# Changelog — MCC Lite

## 2026-04-12 — Post-source-review cleanup

- **Branding rename (filenames):** `scripts/mcp_readonly_evoki.py` → `scripts/mcp_server.py`; `scripts/start_evoki_guardian_stack.bat` → `scripts/start_mcc.bat`; `scripts/stop_evoki_guardian_stack.bat` → `scripts/stop_mcc.bat`; `scripts/stop_evoki_guardian_stack.py` → `scripts/stop_mcc.py`; `MCC_GUARDIAN_LITE.spec` → `MCC.spec`. All internal references in `mcp_guardian_app.py`, `smoke_mcp_server.py`, `start_mcc.bat`, `stop_mcc.bat`, `stop_mcc.py` and `MCC.spec` updated accordingly.
- **Branding rename (display name):** FastMCP server display name `"MCC Lite"` → `"MCC"` so the wire-protocol handshake stays edition-neutral. README still describes Lite as the file-tools edition; only the protocol-level identifier changes.
- **Branding rename (build artifact):** PyInstaller `name="MCC_GUARDIAN_LITE"` → `name="MCC"`, plus header comments in `MCC.spec` updated to reference `mcp_server.py`.
- **BUG-011b (security, info-leak hardening):** In `read_file`, the blocklist check (`_is_blocked_file`) now runs **before** the `is_file()` existence check. Previously a non-existing `.env` returned a different deny reason than an existing-but-blocked `.env`, leaking existence by differential error message. Both paths now return the same `"File blocked by policy"` reason if the name matches the blocklist, regardless of whether the file exists.
- **BUG-018 server strings (i18n):** All ~30 user-facing German strings in `mcp_server.py` migrated to English. This covers `_build_auth_provider` (4 OAuth/OIDC missing-config errors), `_resolve_user_path` (5 path-validation errors), `_run_request_pipeline` (3 deny reasons — IP blocked, too many requests, client blocked), `_check_policy_integrity_or_deny` (4 integrity errors), `read_file`/`write_file`/`create_directory`/`delete_path`/`list_directory`/`search_files` tool exception strings (~12 entries), `_write_allowed` (5 permission strings), and all `print("MCC MCP: …", file=stderr)` startup/runtime messages (~12 statements). **Protocol-level `deny_detail` codes** (snake_case identifiers like `bearer_invalid`, `policy_integrity_mismatch`, `rate_exceeded_*`) were intentionally **not** changed — they are stable parse targets for monitoring tools and have always been ASCII.
- **BUG-017 (cosmetic):** Removed remaining Terminal-MCP wording from `mcp_guardian_app.py`: docstring of `_require_master_password_confirm` now says "sicherheitskritische Aktionen" instead of "Terminal-MCP aktivieren"; integrity-category checkbox in the Advanced tab now reads "Schreib-/Ausführungstools" instead of "Ausführung (Terminal / Skripte)". MCC Lite never registered terminal/script execution tools — these labels were the last cosmetic leakage from earlier editions.
- **BUG-HELP:** `scripts/HELP.md` replaced with an honest English quick-reference stub (~70 lines covering first start, stack startup, tab overview, config files, env vars, locale switching, GitHub link). Full reference remains German-only in `HILFE.md`. `_help_content()` in `mcp_guardian_app.py` is now locale-aware: loads `HELP.md` when `mcc_locale() == "en"`, otherwise `HILFE.md`, with `HILFE.md` as universal fallback. Header line in `HILFE.md` cleaned: "Stand: MCC Control Center" → "Stand: MCC".
- **Example policy hardening (`config/mcp_policy.example.json`):** `permissions.mode` changed from `read_write` to `read_only`; `permissions.write_allow_paths` changed from `["C:\\MeinProjekt"]` to `[]`; the five Extended-edition feature flags (`sqlite_mcp`, `gemini_cli`, `github_mcp`, `git`, `at_mention`) removed entirely; new `_note` field documents how to enable write mode and which fields are reserved for future editions. The example is now consistent with the secure-by-default `DEFAULT_POLICY` baked into `mcp_server.py`.
- **README updates:** Both `README.md` (DE) and `README_EN.md` gained two new sections — *"Reservierte Policy-Felder" / "Reserved policy fields"* listing which policy fields Lite ignores (so users migrating from future editions know what is silently dropped), and *"Herkunft" / "Origin"* explaining MCC's relationship to EVOKI V5 honestly: MCC extracts the file-tool governance primitive as a standalone, independently-licensed tool from EVOKI's broader research. Footer reformulated from "Built with EVOKI V5" / "Erstellt mit EVOKI V5" to "Born in / Entstanden im Forschungsprojekt EVOKI V5".
- **`.gitignore` (additive, null-risk):** Added entries for runtime artifacts that should never enter the public repo: `config/mcp_policy.json`, `config/keystore.enc`, `config/keystore.salt`, `config/blocked_ips.json`, `config/rate_state.json`, `config/guardian_ui.json`, `config/policy_integrity_startup.json`, `config/export_counter.json`, `config/.mcc_first_run_ack`, `logs/`. The staging directory is not a git repo, so no history rewrite was needed; this prepares the file for clean import into the public `mcc` repo.
